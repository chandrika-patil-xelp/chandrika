Support policy

This user guide was created to help you quickly configure our theme. Please read it carefully, it will help you to eliminate most of potential problems with incorrect configuration.

× Important:

please note that as authors of this theme we are able to provide support only for the features which we created and for the issues related strictly to this theme. We do not provide support for OpenCart configuration, maintenance etc.

Questions about OpenCart

× Important:

all available settings and features of this theme are described in chapter "Configure Theme". We are able to provide support only for those features. OpenCart's features are beyond the scope of our support.

Elements such as category management, product management, checkout process, payment methods and many others are default OpenCart functionalities and are beyond the scope of our support. How to get OpenCart support:

  1. If you have a question about OpenCart itself, please ask on OpenCart Forum. Almost every detail of OpenCart configuration is described there so you will find answers for most of your questions. You can also find there many helpful tips and tutorials.
  2. Search in the online OpenCart user guide. You can find there all the basic OpenCart configuration details and other information.

Third-party addons

This theme may not be compatible with some third-party addons. It’s impossible for theme authors to make the theme compatible by default with all existing addons because there are hundreds of available OpenCart addons. Only the “default” theme is compatible with all third-party addons, because all addons are designed to work with the “default” theme.

Generally all addons can work with all themes but sometimes they have to be integrated manually. That’s because sometimes the extension and the theme override the same OpenCart template file so there can be a conflict.

Support

Author of the extension should provide detailed instruction about how to integrate the extension with any custom theme.

Compatibility

with other themes in general or with other themes already installed

Our theme (and any other custom theme) isn't compatible with all other themes by default. We can guarantee correct work of Coolbaby only on clean installation of default OpenCart template with clean database. Our theme in default OpenCart rewrites only 3 core files: - \admin\controller\setting\setting.php
- \admin\controller\setting\store.php
- \admin\view\template\design\layout_form.tpl
Such changes are done in order to make installation process for you in easiest way. You don’t need to add theme’s modules manually - all blocks are added automatically to your database. Other core files aren’t rewritten. If you install Coolbaby on other theme or any other theme on Coolbaby you may see conflicts in work of your site, because other theme can overwrite files of default OpenCart template.

    Troubleshooting brief F.A.Q.

    1. Why I see warning "Permission Denied..." and haven't access to modules or products editing?

    Firstly, check if you use the same version of Coolbaby theme as your default OpenCart version. Nevertheless, to have all permissions: go in admin - system - user - users group - choose Top administrator. Then "check all" for all permissions

    2. Why some category isn't displayed in the top menu?

    To see some category in the top menu:go in admin - Catalog - Categories - category editing page - Tab "Data" check item "top"

    2. I've entered in admin panel module my Facebook name, but it isn't displayed in Facebook widget in Footer info block?

    in Facebook widget you need to enter facebook profile name of your company page. For example, for such test page of not famous company - https://www.facebook.com/pages/Computers/168634786648306, profile name is "pages/Computers/168634786648306". For famous company like ThemeForest, the name is simply "themeforest".

    3. I don't see any rollover effects on product images. How I can set rollover effect?

    While your move mouse across product image your see additional image. So, firstly, you've upload at least one additional image in: Catalog -> Products -> tab Image.
    And secondly, this additional image has be not the same as main image in: Catalog -> Products ->tab Data.
    To see advanced image rollover effect (some previews in product listings) you need to upload some images in Catalog -> Products -> tab Image.
